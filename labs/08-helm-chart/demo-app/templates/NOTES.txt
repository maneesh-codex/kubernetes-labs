     --namespace {{ .Release.Namespace }} --timeout=120s

2. Reach the application:

{{- if .Values.ingress.enabled }}
   An Ingress was created on IngressClass "{{ .Values.ingress.className }}".
{{- range $host := .Values.ingress.hosts }}
{{- range .paths }}
   http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
{{- end }}
{{- end }}

   If that hostname does not resolve, add it to /etc/hosts pointing at your
   ingress controller's address, or curl with an explicit Host header:

     curl -H 'Host: {{ (first .Values.ingress.hosts).host }}' http://localhost/
{{- else if contains "LoadBalancer" .Values.service.type }}
   A LoadBalancer Service was created. Provisioning the external address can
   take a couple of minutes -- watch it with:

     kubectl get svc {{ include "demo-app.fullname" . }} \
       --namespace {{ .Release.Namespace }} -w

   Then:

     export SERVICE_IP=$(kubectl get svc {{ include "demo-app.fullname" . }} \
       --namespace {{ .Release.Namespace }} \
       -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
     curl "http://${SERVICE_IP}:{{ .Values.service.port }}/"
{{- else if contains "NodePort" .Values.service.type }}
   A NodePort Service was created:

     export NODE_PORT=$(kubectl get svc {{ include "demo-app.fullname" . }} \
       --namespace {{ .Release.Namespace }} -o jsonpath='{.spec.ports[0].nodePort}')
     export NODE_IP=$(kubectl get nodes \
       -o jsonpath='{.items[0].status.addresses[?(@.type=="InternalIP")].address}')
     curl "http://${NODE_IP}:${NODE_PORT}/"
{{- else }}
   The Service is ClusterIP, so port-forward to reach it from your laptop:

     kubectl port-forward --namespace {{ .Release.Namespace }} \
       svc/{{ include "demo-app.fullname" . }} 8080:{{ .Values.service.port }}

   Then in another terminal:

     curl http://localhost:8080/
     curl http://localhost:8080/api/info
     curl http://localhost:8080/metrics
{{- end }}
